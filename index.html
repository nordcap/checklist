A) Установка GIT - top

Система контроля версий - неотьемлемый и даже необходимый инструмент разработки приложения. Если использовать bower то ставить обязательно. Install git

Основные команды git config --global user.name "Alex Budaev" //обязательное указание имени
git config --global user.email johndoe@example.com // обязательное указание email
git help "команда" //открыть руководство по команде "команда"
git init //создание репозитория
git status //статус репозитория
git add . (git add --all)//добавить файлы для индексации
git commit - m "there comment" //коммит
git remote add origin https://github.com/path_repository //добавить удаленный репозиторий
git push -u origin master //запушить в удаленный репозиторий
git pull origin master //вытянуть изменения в ветку мастер
git clone https://github.com/path_repository //клонировать удаленный репозиторий
git brunch //список веток
git brunch nameBrunch // создать новую ветку
git checkout nameBrunch // переключиться на другую ветку
git log --graph --all // вывод списка коммитов
git merge nameBrunch// сливает ветку с мастером

В файл исключений .gitignore добавить node_modules
app/bower_components
app/sass-cache //если используется compass

B) Установка пакетных менеджеров npm, bower - top

  1. Установка node.js Требуется для работы npm и bower
  2. Установка npm глобально в системе $ npm install npm -g
  3. Установка bower глобально в системе $ npm install -g bower
  4. Создать файл package.json (или скопировать из другого проекта) npm init
  5. Создать файл bower.json (или скопировать из другого проекта) bower init
  6. Установить необходимые модули npm либо поочередно npm install --save-dev namePackage либо используя зависимости, указанные в файле package.json npm install
  7. Установить необходимые модули bower либо поочередно $ bower install --save namePackage либо используя зависимости, указанные в файле bower.json bower install
  8. Папку bower_components можно перенести в папку разработки app. Для корректного нахождения компонентов нужно создать файл .bowerrc с таким содержанием 
    {
  "directory": "app/bower_components/"
}
  9. Общая структура приложения 
                        [.git]
                    [app]
                        [bower_components]
                    [dist]
                    [node_modules]
                        .bowerrc
                        .gitignore
                        .htaccess
                        bower.json
                        gulpfile.js
                        package.json

С) Сборщик проектов Gulp - top

  1. Установить сборщик проектов Gulp глобально npm install --global gulp
  2. Установить Gulp локально в проект (Если его нет в package.json) npm install --save-dev gulp
  3. Создать файл gulpfile.js (или скопировать из другого проекта)
  4. Задачи запускаются так gulp nameTask
  5. Особенность использования плагина gulp-useref.

    CSS файлы и JS файлы обрамляются комментариями в которых указаны файлы назначения. Данный плагин "склеивает" обрамленные файлы и переименовывает.

    //склейка css файлов
    build:css css/vendor.min.css
    link rel="stylesheet" href="css/font-awesome.css"
    link rel="stylesheet" href="css/font-awesome.css"
    endbuild

    //склейка js файлов
    build:js js/vendor.min.js
    script src="bower_components/jquery/dist/jquery.js" script
    script src="bower_components/materialize/bin/materialize.js" script
    endbuild
  6. Особенность использования плагина gulp-wiredep.

    Плагин ищет в файлах комментарии которые позволяют подключить компоненты bower без ручного указания пути.

    bower:js
    script src="bower_components/jquery/dist/jquery.js" script
    endbower

D) Установка compass - top

  1. Compass требует наличия Ruby. Установить с оф.сайта
  2. Установить compass в папке проекта. Обязательно следует настроить конфигурацию в файле config.rb gem update --system
    gem install compass
    compass create app --bare //чистый проект

    compass create app
    --sass-dir "sass"
    --css-dir "css"
    --javascripts-dir "js"
    --images-dir "img"

E) Установка susy - top

Установить susy в системе gem install susy

Подключить susy в проект можно несколькими способами

  1. Если используется с compass compass install susy //текущий проект
    compass create --using susy app //при создании нового проекта (app-папка название проекта см.выше)
  2. Отдельно через bower bower install susy --save

Если используется отдельно от compass, то в scss файле подключаем @import "bower_components/susy/sass/susy";

G) Структура папки ./app/sass - top

Используется архитектура 7-1: 7 папок, 1 файл main.scss, который импортирует все остальные файлы из других папок.

[base]
[components]
[layout]
[pages]
[themes]
[utils]
[vendors]
    main.scss

  • Папка Base

    Папка base/ содержит то, что мы можем назвать общим шаблоном проекта. Там вы можете найти файл сброса, некоторые типографские правила, и, вероятно, стили (я привык их называть _base.scss), определяющие некоторые стандартные стили для часто используемых элементов HTML.

  • Папка Layout (или Partials)

    Папка layout/ содержит всё, что принимает участие в постройке раскладки сайта или приложения. Эта папка может содержать стили для основных частей сайта (шапка, подвал, навигация, боковая панель…), сетка или даже CSS-стили для всех форм.

  • Папка Components (или modules)

    Для маленьких компонентов есть папка components/. В то время, как layout/ – основные (определяют общий каркас), код внутри components/ больше сфокусирован на виджетах и содержит все модули, вроде слайдера, загрузчика и тому подобных виджетов. Обычно файлов в components/ много, если приложение или сайт состоит из множества мелких модулей.

  • Папка Pages

    Если у вас есть стили, зависящие от страницы, лучше положить их в папку pages/, в файл, названный в честь страницы. Например, не редкость – иметь очень конкретные стили для главной страницы, следовательно, существует потребность в _home.scss в pages/.

  • Папка Themes (Это очень зависит от проекта и не сильно распространено.)

    В больших сайтах и проложениях не редко есть разные темы оформления. Есть разные способы работы с темами, я лично предпочитаю складывать их в папку themes/.

  • Папка Utils

    Папка utils/ собирает все инструменты и помощники Sass в проекте. Каждая глобальная переменная, функция и примесь должна быть помещена сюда.

    Папка utils/ может также быть названа helpers/, sass-helpers/ или sass-utils/, на ваше усмотрение.

  • Папка Vendors

    Иногда некоторые проекты (которые не используют Bower, Gulp) будут иметь папку vendors/, содержащую все CSS-файлы из внешних библиотек и фреймворков – Normalize, Bootstrap, jQueryUI, FancyCarouselSliderjQueryPowered и так далее.

    Если вы хотите что-то переписать в файлах данной папки, то я рекомендую вам ввести восьмую папку vendors-extensions/, в которой вы будете хранить файлы перезаписи свойств с точно такими же именами.

    Например, файл vendors-extensions/_boostrap.scss, содержащий все CSS-правила на перезапись CSS-правил Bootstrap. Это для того, чтобы не править сами внешние модули, что на самом деле не очень хорошая идея.

[sass]
|
|– [base]
|   |– _reset.scss       # Reset/normalize
|   |– _typography.scss  # Типографские правила
|   |- _base.scss        # Переопределение стандартных стилей
|
|– [components]
|   |– _buttons.scss     # Кнопки
|   |– _carousel.scss    # Карусель
|   |– _cover.scss       # Обложка
|   |– _dropdown.scss    # Выпадающий список
|   ...                  # и т.д.
|
|– [layout]
|   |– _navigation.scss  # Навигация
|   |– _grid.scss        # Сетка
|   |– _header.scss      # Шапка
|   |– _footer.scss      # Подвал
|   |– _sidebar.scss     # Боковая панель
|   |– _forms.scss       # Формы
|   ...                  # и т.д.
|
|– [pages]
|   |– _home.scss        # Стили, особые для главной страницы
|   |– _contact.scss     # Стили, особые для страницы контактов
|   ...                  # и т.д.
|
|– [themes]
|   |– _theme.scss       # Тема по умолчанию
|   |– _admin.scss       # Тема админа
|   ...                  # и т.д.
|
|– [utils]
|   |– _variables.scss   # Переменные Sass
|   |– _functions.scss   # Функции Sass
|   |– _mixins.scss      # Примеси Sass
|   |– _helpers.scss     # Помощники классов & placeholder’ов
|
|– [vendors] (Если не используется bower)
|   |– _bootstrap.scss   # Bootstrap
|   |– _jquery-ui.scss   # jQuery UI
|   ...                  # и т.д.
|
|
`– main.scss             # главный файл Sass

H) Файл main.scss - top

Главный файл (обычно названный main.scss) должен быть единственным файлом Sass, который не начинается с нижнего подчеркивания. Этот файл не должен содержать ничего, кроме @import и комментариев.

Файлы должны быть импортированы в соответствии с папкой размещения, один за другим, в соответствуещем порядке:

  1. vendors/
  2. utils/
  3. base/
  4. layout/
  5. components/
  6. pages/
  7. themes/
@import 'vendors/bootstrap';
@import 'vendors/jquery-ui';

@import 'utils/variables';
@import 'utils/functions';
@import 'utils/mixins';
@import 'utils/placeholders';

@import 'base/reset';
@import 'base/typography';

@import 'layout/navigation';
@import 'layout/grid';
@import 'layout/header';
@import 'layout/footer';
@import 'layout/sidebar';
@import 'layout/forms';

@import 'components/buttons';
@import 'components/carousel';
@import 'components/cover';
@import 'components/dropdown';

@import 'pages/home';
@import 'pages/contact';

@import 'themes/theme';
@import 'themes/admin';
